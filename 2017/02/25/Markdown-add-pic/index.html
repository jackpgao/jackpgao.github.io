Markdown方便的加图工具

发表于 2017-02-25 | 分类于 效率

问题背景

  • 贴图在markdown编辑器上比较麻烦。
    但是，可以指定图片url，因此有一个好的图床就成了开心写文档的前提。

免费图床

  1. 使用新浪微博作为图床。
    手动上传：也就是发微博，但是会设置成只自己可见的微博。
    我会，去掉水印，点大图-原图-复制图片链接。比较麻烦。
    好奇为啥微博图片没有做防盗链?
    插件上传：围脖是个好图床
    chrome插件
    这个插件我也没用过，但是看好多地方推荐，应该还不错。

  2. 使用花瓣
    优点：花瓣的插件非常好用，直接能识别网页的图片，并采集到自己的画板中。
    缺点：有防盗链，默认进去不是大图，需要点一下，再复制出图片链接。
    单是防盗链这一点，就已经给毙了。

  3. 使用七牛云存储
    用户有10G免费的空间可用，可以设置对外访问。
    上传：手动上传，一张一张复制图片链接，比较麻烦。
    插件1：极简图床
    插件2：七牛在线存图
    问题：
    极简图床能够直接生产插入markdown的图片连接，但是，图片上传的方式有点累，只能开一个插件的页面，本地上传或者复制图片连接。如果能像花瓣那样直接针对网页图片有一个识别的按钮，并自动生成图片连接就好了。

掏钱的方案

说了这么多，其实有一个非常好用，但是有点贵的方案。
http://zh.mweb.im/

MWeb
关于Mweb就不介绍了，直接看连接就行。

有人说MWeb是Mac平台上，最好用的Markdown编辑器，高级版自带了上述图床配置，配置好七牛的各种Key，图片直接拖入到编辑器即可。
另外，好像还支持静态网页导出。
售价￥68，没去试。

本次方案

下面说一下，比较屌丝，但是也能解决问题的免费方案。
感谢开源。

先上链接：
markdown图片实用工具@github

作者博客

Alfred Powepack 2.6.dmg

操作方式，都在作者的github-wiki了，需要下载的Alfred Powerpack也提供了下载包。
导入Alfred后，需要做如下设置：

  • 设置快捷键

  • 设置七牛的各种key和域名。Alfred中直接呼出mdimgsetup，或者找到导入项目的目录，直接改对应的ini配置文件。

使用感受：

  • 能够对截图工具截的图直接上传，对于snip(腾讯提供的一个截图工具，比较推荐)在retina下截的图，能够生成一个html的图片标记，而不是直接给一个七牛的图片链接。这样图片的size就有了保证，很赞。
  • 对于选中-复制-快捷键上传的图片，会直接放一个七牛的链接到粘贴板，直接command+v粘贴就行了。这里有个建议，能不能直接生成![](url)这种方式，回头看看相关的代码，改一下，应该比较简单。

总结

  • 其实，如果是每天用Markdown，花个几十块钱买一个软件，并不是多贵的事情，反而省去了这些折腾的步骤，把注意力集中到事情本身。
  • 免费的方案虽然折腾，但是整个过程下来，虽然遇到了很多问题，但是充满了乐趣。

Update

  • 目前该工具不支持gif上传，默认的图片会进行压缩，统一为png格式，已经联系了作者，作者表示会进行更新。作者已经更新。

  • 后来我买了Mweb，因为实在是太方便了

    • 拒绝折腾，把时间都集中到创作里去吧！
